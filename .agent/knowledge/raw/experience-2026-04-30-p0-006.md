## 3. 修复模式总结

### 3.1 接口属性必选化

**场景**: 接口中可选属性与实际使用不一致

**解决方式**: 将可选属性改为必选

```typescript
// 修复前
interface Command {
  name: string;
  subcommands?: Command[];
}

// 修复后
interface Command {
  name: string;
  subcommands: Command[];
}
```

### 3.2 索引签名模式

**场景**: 测试中需要动态访问对象属性

**解决方式**: 添加索引签名

```typescript
// 修复前
interface Person {
  name: string;
  age: number;
}

// 修复后
interface Person {
  name: string;
  age: number;
  [key: string]: any;
}
```

### 3.3 测试导入补全

**场景**: 测试文件缺少必要的测试框架导入

**解决方式**: 添加对应的导入语句

```typescript
import { beforeEach, describe, expect, it } from 'vitest';
```
